What is an AI Agent?

Core Characteristics:

  • Autonomous System: Can perceive environment and take actions
  • Goal-Oriented: Works towards specific objectives
  • Decision-Making: Uses reasoning to choose appropriate actions
  • Tool Usage: Can interact with external systems and APIs
  • Memory & State: Maintains context across interactions

Key Capabilities:

  • Reactivity & Proactivity
  • Social Ability
  • Planning and Reasoning
  • Environment Interaction
  • Adaptive Learning
AI agents represent a paradigm shift from simple chatbots to sophisticated systems that can plan, reason, and execute complex tasks autonomously.

LangGraph Overview

Graph-Based Agent Orchestration

Core Concepts:

  • State-Based Architecture: Persistent state across interactions
  • Graph Structure: Nodes (functions) and edges (transitions)
  • Conditional Routing: Dynamic flow control
  • Checkpointing: Built-in persistence and recovery

Key Features:

  • Supervisor Pattern: Centralized coordination
  • Swarm Orchestration: Distributed collaboration
  • Human-in-the-Loop: Interactive workflows
  • Streaming Support: Real-time updates
LangGraph takes a graph-theoretical approach to agent orchestration, providing explicit state management and flexible routing for complex, long-running workflows.

OpenAI ADK Overview

Native OpenAI Integration

Design Philosophy:

  • Native Integration: Built for OpenAI models
  • Simplicity First: Minimal boilerplate
  • Production Ready: Enterprise-grade features
  • Type Safety: Full TypeScript support

Key Features:

  • Assistant API Integration: Seamless usage
  • Tool Calling: Native function calling
  • Session Management: Built-in conversation handling
  • Deployment Tools: Production utilities
OpenAI ADK prioritizes developer experience and production readiness with tight integration to OpenAI's ecosystem and minimal complexity.

Feature Comparison: Creating an Agent

LangGraph

import { createReactAgent } from "@langchain/langgraph/prebuilt";

const agentTools = [new TavilySearchResults({ maxResults: 3 })];
const agentModel = new ChatOpenAI({ temperature: 0 });

// Initialize memory to persist state
const agentCheckpointer = new MemorySaver();
const agent = createReactAgent({
  llm: agentModel,
  tools: agentTools,
  checkpointSaver: agentCheckpointer,
});

OpenAI ADK

import { Agent } from 'openai-agents';

const agent = new Agent({
  name: 'MyAgent',
  instructions: 'You are a helpful assistant',
  model: 'gpt-4',
  tools: [weatherTool, calculatorTool]
});
Key Difference: LangGraph requires explicit state definition and graph construction for maximum control. OpenAI ADK offers straightforward approach with sensible defaults.

Feature Comparison: Creating a Tool

LangGraph Tool

import { tool } from "@langchain/core/tools";

const weatherTool = tool(
  async ({ location }) => {
    const response = await fetch(`/api/weather/${location}`);
    return response.json();
  },
  {
    name: "get_weather",
    description: "Get current weather for a location",
    schema: z.object({
      location: z.string().describe("City name")
    })
  }
);

OpenAI ADK Tool

const weatherTool = {
  type: 'function',
  function: {
    name: 'get_weather',
    description: 'Get current weather for a location',
    parameters: {
      type: 'object',
      properties: {
        location: { type: 'string', description: 'City name' }
      },
      required: ['location']
    }
  },
  handler: async ({ location }) => {
    const response = await fetch(`/api/weather/${location}`);
    return response.json();
  }
};

Feature Comparison: Agent Invocation

LangGraph Invocation

// Simple invocation
const result = await app.invoke({
  messages: [new HumanMessage("What's the weather in Paris?")]
});

// Streaming with state updates
const stream = await app.stream({
  messages: [new HumanMessage("Plan a trip to Tokyo")]
});

for await (const chunk of stream) {
  console.log(chunk);
}

OpenAI ADK Invocation

// Simple invocation
const response = await agent.run({
  message: "What's the weather in Paris?",
  sessionId: "user-123"
});

// Streaming response
const stream = await agent.stream({
  message: "Plan a trip to Tokyo",
  sessionId: "user-123"
});

for await (const chunk of stream) {
  console.log(chunk.content);
}

Multi-Agent Orchestration

LangGraph Patterns

Supervisor Pattern:

const supervisor = createSupervisor({
  agents: [flightAssistant, hotelAssistant],
  llm,
  prompt: "You manage assistants. Assign work to them, one at a time."
}).compile();

Swarm Pattern:

const swarm = createSwarm({
  agents: [flightAssistant, hotelAssistant],
  defaultActiveAgent: "flight_assistant",
})

OpenAI ADK Pattern

const triageAgent = new Agent({
  name: 'triage_agent',
  instructions: 'Handoff to the appropriate agent based on the language of the request.',
  handoffs: [spanishAgent, englishAgent],
});

const result = await run(triageAgent, 'Write a haiku about recursion in programming.');

Human-in-the-Loop Capabilities

LangGraph Approach

// Add interrupt for human review
workflow.addNode("human_review", async (state) => {
  return { requires_approval: true };
});

// Conditional edge with human approval
workflow.addConditionalEdges(
  "agent_action",
  (state) => state.requires_approval ? "human_review" : "continue"
);

// Resume after approval
const result = await app.invoke(input, config);
await app.invoke({ approval: true }, config);

OpenAI ADK Approach

const agent = new Agent({
  name: 'approval_agent',
  instructions: 'Process requests with human approval',
  model: 'gpt-4',
  humanApprovalRequired: true
});

const response = await agent.run({
  message: "Transfer $1000 to account 123",
  sessionId: "user-123",
  onApprovalRequired: async (context) => {
    return await requestHumanApproval(context);
  }
});

Agent Evaluation and Observability

LangGraph Observability

// LangSmith integration
import { LangSmith } from "langsmith";

const tracer = new LangSmith({
  projectName: "agent-evaluation"
});

// Trace agent execution
const app = workflow.compile({
  callbacks: [tracer]
});

// Custom metrics
workflow.addNode("metrics", async (state) => {
  await tracer.log({
    event: "tool_usage",
    metadata: { tool: state.current_tool }
  });
  return state;
});

OpenAI ADK Observability

// Built-in monitoring
const agent = new Agent({
  name: 'monitored_agent',
  model: 'gpt-4',
  monitoring: {
    enabled: true,
    metrics: ['latency', 'token_usage', 'success_rate'],
    alerts: {
      errorRate: { threshold: 0.1 },
      latency: { threshold: 5000 }
    }
  }
});

// Access metrics
const metrics = await agent.getMetrics({
  timeRange: '24h'
});

OpenAI ADK Handoff

const billingAgent = new Agent({ name: 'Billing agent' });
const refundAgent = new Agent({ name: 'Refund agent' });

const triageAgent = Agent.create({
  name: 'Triage agent',
  handoffs: [billingAgent, handoff(refundAgent)],

When to Choose Each Framework

Choose LangGraph When:

  • Complex, multi-step workflows required
  • Fine-grained state management needed
  • Custom orchestration patterns
  • Long-running, stateful processes
  • Research and experimentation focus

Choose OpenAI ADK When:

  • Rapid prototyping and development
  • Native OpenAI model integration preferred
  • Production deployment is priority
  • Limited ML engineering resources
  • Standard agent patterns sufficient

Trade-offs Analysis:

  • Flexibility vs Integration: LangGraph offers more flexibility; ADK provides tighter integration
  • Learning Curve: LangGraph steeper; ADK more accessible
  • Community: LangGraph has larger community; ADK has official support
  • Production Readiness: ADK more production-ready; LangGraph more experimental
